css命名规则

发布时间：2016-12-25 标签:css

css在日常工作当中用得虽然比较多，但是很多细节却不是十分清楚，最近一段时间在学习其他的一些东西，好久都没有学习css的相关知识了，今天抽空学习一下。

1.css命名规范

首先想说一下选择器的命名规范，平常工作当中基本都是使用小些字母来命名，但是其命名的字符范围包括：数字(0~9)，字母(a-zA-z)，下划线(_)，连字符(-)，还有其他转义字符，对于首字符也有要求，不能是或者连字符+数字开头，至少一个字符，现在动手写个demo看一下效果，以谷歌浏览器为测试工具：

<p class="123abc">class is 123abc</p>
<p class="-123">class is -123</p>
<p class="-">class is -</p>
<p class="_123">class is _123</p>
<p class="abc123">class is abc123</p>

对上面的所有class添加了字体颜色为红色样式，可以看出前三个都是没有正确识别的，因此在实际使用过程当中尽量使用字母为开头字符会比较好，下划线用得就比较少，但是如果想使用其他字符，就需要转义了，还是上面的实例，稍作如下修改即可使class="123abc"的样式得到匹配:

 .\31 23abc{
   color:red;
 }

由于命名规范的原因，因此不能直接使用数字1作为开头，但是使用1对应的unicode编码就可以，这里需要注意的是unicode编码后面需要添加空格。

在其他编程语言当中，也会遇到字符转义的问题，为什么需要转义，有两个原因：1.有些字符已经用作表示其他意思了，作为了特殊的字符，如果需要使用它们本身的符号，就需要转义，比如在css当中，[!.,:'"{}#|~()*=&%]等这些特殊字符，它们往往用作css语法规范与定义，因此如果要使用它们本来的意思，就需要转义；2.使用转义字符来表示字符集中定义的字符，比如车换行空格等字符，这些字符都没有现成的文字代号。所以只能用转义字符来表示。基于上述原因，在css中使用特殊字符:的时候，我们添加\进行转义，或者使用unicode编码，具体实例如下：

.\:a1{
  color: #fff;
}
.\3a a1{
  background-color: blue;
}
<p class=":a1">class is :a1</p>
